# Quick Start

This guide will help you get started with wit in just a few minutes.

## Prerequisites

- Node.js 22.13.0 or higher
- npm or yarn

## Installation

```bash
# Clone and build wit
git clone https://github.com/abhiaiyer91/wit.git
cd wit
npm install
npm run build
npm link   # Makes 'wit' available globally
```

## Your First Repository

### Initialize a Repository

```bash
mkdir my-project
cd my-project
wit init
```

You'll see:
```
Initialized empty wit repository in /path/to/my-project/.wit
```

### Add and Commit Files

```bash
# Create a file
echo "# My Project" > README.md

# Stage the file
wit add README.md

# Commit with a message
wit commit -m "Initial commit"
```

### Check Status

```bash
wit status
```

### View History

```bash
wit log
```

## Essential Commands

### Stage and Commit in One Step

```bash
# Stage all tracked files and commit
wit commit -a -m "Update everything"
```

### Quick WIP Save

```bash
# Save work-in-progress with auto-generated message
wit wip -a
```

### Switch Branches

```bash
# Create and switch to a new branch
wit switch -c feature-branch

# Switch back to main
wit switch main
```

<Note>
wit automatically saves your uncommitted changes when switching branches, so you never lose work!
</Note>

### Undo Mistakes

```bash
# Undo the last operation (commit, branch creation, etc.)
wit undo

# Undo the last 3 operations
wit undo --steps 3
```

## Visual Interfaces

### Web UI

```bash
wit web
```

Opens a modern dashboard at `http://localhost:3847` with:
- Visual commit graph
- Side-by-side diffs
- One-click staging
- Search across commits and files

### Terminal UI

```bash
wit ui
```

Interactive terminal interface with keyboard navigation.

### ASCII Graph

```bash
wit graph
```

```
● a1b2c3d4 (main) Latest commit - Alice, today
● e5f6g7h8 Add feature - Bob, yesterday
● i9j0k1l2 Initial commit - Alice, last week
```

## Working with GitHub

### Login to GitHub

```bash
wit github login
```

This opens a browser to authenticate. Once logged in, your credentials are stored securely.

### Clone a Repository

```bash
wit clone https://github.com/user/repo.git
cd repo
```

### Push Changes

```bash
# Make changes
echo "update" >> README.md
wit add .
wit commit -m "Update readme"

# Push to GitHub
wit push origin main
```

### Pull Changes

```bash
wit pull
```

<Tip>
wit uses your GitHub profile for commit author information automatically after login.
</Tip>

## AI Features (Optional)

Enable AI-powered features for commit message generation, code review, and more.

### Setup

```bash
# Set your OpenAI or Anthropic API key
export OPENAI_API_KEY=sk-your-key-here
# OR
export ANTHROPIC_API_KEY=sk-ant-your-key-here

# Verify setup
wit ai status
```

### Use AI

```bash
# Natural language commands
wit ai "what files have I changed?"

# Generate commit message from staged changes
wit ai commit

# Stage all, generate message, and commit
wit ai commit -a -x

# Get code review
wit ai review
```

## Self-Hosted Platform (Optional)

Run your own Git server with pull requests and issues.

### Start a Server

```bash
# Start wit server
wit serve --port 3000 --repos ./my-repos
```

### Push to Your Server

```bash
# Add remote to your project
wit remote add origin http://localhost:3000/myuser/myproject.git
wit push -u origin main
```

### Pull Requests & Issues

```bash
# Create a pull request
wit pr create -t "Add feature"

# List PRs
wit pr list

# Create an issue
wit issue create "Bug: Login fails"

# List issues
wit issue list
```
