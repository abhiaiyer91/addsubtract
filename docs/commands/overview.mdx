## Quick Reference

### Most Used Commands

| Command | Description |
|---------|-------------|
| `wit init` | Create a new repository |
| `wit add <files>` | Stage files for commit |
| `wit commit -m "msg"` | Create a commit |
| `wit status` | Show working tree status |
| `wit log` | Show commit history |
| `wit switch <branch>` | Switch branches |
| `wit undo` | Undo the last operation |
| `wit wip` | Quick WIP commit |

### wit-Exclusive Commands

These commands are unique to wit and don't exist in Git:

| Command | Description |
|---------|-------------|
| `wit undo` | Undo any operation (commit, branch, merge, etc.) |
| `wit wip` | Create WIP commit with auto-generated message |
| `wit amend` | Simpler interface for amending commits |
| `wit uncommit` | Undo commits, keep changes staged |
| `wit cleanup` | Find and remove merged/stale branches |
| `wit stats` | Repository statistics and insights |
| `wit snapshot` | Quick checkpoints without full commits |
| `wit scope` | Limit operations to specific paths (monorepo) |
| `wit history` | Show operation history (what undo uses) |

## Command Syntax

```bash
wit <command> [options] [arguments]
```

### Common Options

Most commands support these options:

| Option | Description |
|--------|-------------|
| `--help`, `-h` | Show help for a command |
| `--version`, `-v` | Show wit version |

### Getting Help

```bash
# General help
wit --help

# Help for specific command
wit commit --help
wit undo --help
```

## Differences from Git

wit aims to be more intuitive than Git:

| Git | wit | Improvement |
|-----|-------|-------------|
| `git checkout <branch>` | `wit switch <branch>` | Clear intent |
| `git checkout -- <file>` | `wit restore <file>` | Clear intent |
| `git commit --amend` | `wit amend` | Simpler syntax |
| `git reset HEAD~1` | `wit uncommit` | Clear intent |
| `git stash` (manual) | Auto-stash on switch | Automatic |
| `git reflog` (cryptic) | `wit undo` | Simple undo |
